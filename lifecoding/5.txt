리액트는 사용자 정의 태그를 만드는 기술이다.
이것이 리액트의 본질이다.
< 4. 컴포넌트 만들기 > 에서 리액트에서 어떻게 사용자 정의 태그를 만드는가 라는것을 살펴본다.

사용자 정의 태그를 만들고 싶다면 함수를 정의하면 된다.

function App() {
  return (
    <div>
      <header>
        <h1>
         <a href="/">WEB</a>
        </h1>
      </header>
    </div>
  );
}

👇🏻

function Header() {
  // 사용자 정의태그는 대문자로 시작해야한다
  return (
    // 이 함수는 return값으로 html 코드를 return하면 된다
    <header>
      {/* home으로 이동하는 header영역 */}
      <h1>
        <a href="/">WEB</a>
      </h1>
    </header>
  );
}

function App() {
  return (
    <div>
      <Header></Header>
    </div>
  );
}


------------------------------------------------------------

리액트에서는 사용자 정의태그라는 표현을 쓰지 않고 컴포넌트라는 표현을 쓴다.
즉, 위에서 만든게 컴포넌트를 만든거다.

function App() {
  return (
    <div>
      <Header></Header>
      <Nav></Nav>
      <Article></Article>
    </div>
  );
}

이렇게 다 컴포넌트로 바꿨다.
=> 1. 이전 코드랑 비교하면 훨씬 더 간결해 졌고 
   2. 각각의 코드가 이름을 갖고 있기 때문에 어떠한 취지의 코드인지도 금방 파악할 수 있다.
   3. 내용을 바꾸면 해당 컴포넌트를 사용하고 있는 모든 곳에서 수정이된다. 

=> 이게 리액트의 본질이다.
이렇게 컴포넌트를 만드는 기술인 리엑트 덕분에 여러태그들을 하나의 독립된 부품으로 만들 수 있게 됐다. 
그리고 그 부품을 이용하면 더 적은 복잡도로 소프트웨어를 만들 수 있게 된다. 
동시에 내가 만든 컴포넌트를 다른사람에게 공유할 수도 있고 다른 사람이 만든 컴포넌트를 내가 사용할 수 있게 함으로써 생산성을 획기적으로 끌어올리는 중요한 역할을 하고있고 거대한 리액트 생태계가 존재하고 있는것이다.
